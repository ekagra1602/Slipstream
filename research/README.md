# Research Agent System

This folder defines a multi-agent research workflow with:
- 1 master research agent (orchestrator)
- 5 specialist agents (`empiricalist`, `theorist`, `contrarian`, `falsifier`, `agent-ai-optimizer`)

## Structure
- `master-research-agent.md`: Master agent role, workflow, and synthesis format.
- `agents/empiricalist.md`: Evidence-first investigator.
- `agents/theorist.md`: Model and mechanism builder.
- `agents/contrarian.md`: Counter-position challenger.
- `agents/falsifier.md`: Test designer for disconfirmation.
- `agents/agent-ai-optimizer.md`: Agentic AI workflow optimization specialist.
- `cross_item_transfer_web_agents.md`: DomBot-specific cross-item transfer research brief and benchmark protocol.
- `benchmark_optimization_meta_analysis.md`: Full benchmark evidence synthesis using the 4-agent workflow.

## Operating Model
1. Master defines the research question and constraints.
2. Master briefs all five specialist agents in parallel.
3. Specialists return findings in their required format.
4. Master resolves conflicts and issues a final research brief.

## Deliverable Standard
Every final output from the master should include:
- Claim summary
- Evidence quality assessment
- Competing explanations
- Falsification path
- Decision and confidence
